## 任务一：添加缺失的功能提示（3-5分钟）

### 场景
老师发现非管理员访问管理后台时被静默重定向，没有提示信息。

### 问题
路由守卫设置 `?noPermission=true` 参数但没有任何组件读取它。同样 `?needLogin=true` 也没有被读取。

### 修改文件
`dong-medicine-frontend/src/App.vue`

### 具体步骤
1. 找到 `<script setup>` 中已有的 `const route = useRoute()`
2. 在已有的 `watch` 或 `onMounted` 附近添加：

```js
// 监听路由参数：无权限提示
watch(() => route.query.noPermission, (val) => {
  if (val === 'true') {
    ElMessage.warning('暂无权限访问该页面')
    router.replace({ query: {} })
  }
})

// 监听路由参数：未登录自动弹出登录框
watch(() => route.query.needLogin, (val) => {
  if (val === 'true') {
    showLoginDialog.value = true
    router.replace({ query: {} })
  }
})
```

### 讲解要点
"这是一个防御性编程的改进。之前路由守卫虽然正确地拒绝了未授权访问并重定向，但用户看不到任何反馈。通过watch监听路由参数的变化，在重定向后自动弹出提示或登录对话框，提升了用户体验。"

## 任务四：新增接口限流端点（10分钟）

### 场景
老师问"如果要给某个新的接口加限流，怎么做？"

### 操作
在已有的Controller方法上添加`@RateLimit`注解即可。

### 示例
假设要给问答列表接口添加限流（限制每秒20次）：

```java
// 在 QaController.java 的 list 方法上添加注解
@RateLimit(value = 20, key = "qa_list")
@GetMapping("/list")
public R list(...) { ... }
```

### 讲解要点
"限流功能通过自定义@RateLimit注解实现AOP切面拦截，使用Redis Lua脚本的滑动窗口计数算法，Redis不可用时自动降级为本地令牌桶。只需要一行注解就能给任何接口添加限流，不需要修改业务代码，体现了AOP面向切面编程的优势。"

### 如果老师追问限流原理，展开讲：

```
请求 → RateLimitAspect.around()
  → 解析注解的value(限流次数)和key(限流维度)
  → 拼接key: "ClassName.methodName:userId"
  → 尝试Redis: eval(Lua脚本, key, 限流次数, 窗口秒数)
  → Redis可用: Lua脚本原子执行 INCR + EXPIRE，超过限流返回429
  → Redis不可用: 降级到本地令牌桶，容量100令牌，每秒补充1个
```

## 任务五：添加新的管理后台CRUD（15分钟）

### 场景
老师说"现在如果要新增一个数据表和对应的管理功能，你怎么快速实现？"

### 操作流程（以"药材产地管理"为例）

**第一步：建表**
```sql
CREATE TABLE origins (
    id INT AUTO_INCREMENT PRIMARY KEY,
    name VARCHAR(100) NOT NULL,
    province VARCHAR(50),
    description TEXT,
    created_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP,
    updated_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP
);
```

**第二步：后端**
```java
// 1. Entity
@Data @TableName("origins")
public class Origin extends BaseEntity {
    private String name;
    private String province;
    private String description;
}

// 2. Mapper
@Mapper
public interface OriginMapper extends BaseMapper<Origin> {}

// 3. Service接口
public interface OriginService extends IService<Origin> {}

// 4. Service实现
@Service
public class OriginServiceImpl extends ServiceImpl<OriginMapper, Origin>
    implements OriginService {}

// 5. AdminController中添加端点
// 在 AdminContentController 中添加 CRUD 方法
```

**第三步：前端**
```js
// 在 adminUtils.js 中添加配置
origins: {
  title: '药材产地',
  columns: [
    { prop: 'name', label: '产地名称' },
    { prop: 'province', label: '省份' },
    { prop: 'description', label: '描述' }
  ]
}
```

### 讲解要点
"本平台的管理后台采用了统一的CRUD架构——后端Entity + Mapper + Service三层自动生成基础CRUD，AdminController使用BeanUtils.copyProperties进行DTO到实体的转换；前端使用统一的AdminDataTable组件和配置化的列定义，新增一个管理模块只需要添加配置项，不需要写新的页面组件。这种设计大幅提升了可维护性和扩展性。"

## 通用应对策略

### 如果15分钟内完不成怎么办？
- 先完成核心逻辑（让功能跑通），样式可以后补
- 边写边讲解思路，展示你的编程思维过程
- 如果卡住了，主动说"这里我记得有XX工具方法，让我确认一下"，然后查看文件

### 如何展示工程素养？
- 写代码时主动提到："这里需要考虑XSS安全，所以用DOMPurify过滤"
- 提到边界情况："需要处理null值和空字符串"
- 提到一致性："保持和项目中其他组件一致的写法"

### 最坏情况：完全不会的题目
- "这个功能我之前没有实现过，让我先分析一下需要修改哪些文件"
- 然后打开项目结构，展示你对代码组织的理解
- 即使写不出来，展示分析过程也能得分
