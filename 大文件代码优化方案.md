## 优化方案

### 方案一：巨石 Vue 组件拆分（影响最大）

#### 1.1 PersonalCenter.vue (1131行 → 目标 <400行)

这是项目中最大的单文件组件，模板183行 + 脚本340行 + 样式540行。

**拆分策略：将 7 个功能模块提取为独立子组件：**

```
src/views/PersonalCenter.vue                  (~150行，仅做容器+路由编排)
src/views/personal-center/
  ├── ProfileSection.vue                      (~100行，头像+用户信息+快捷入口)
  ├── StatsDashboard.vue                      (~120行，学习统计面板)
  ├── FavoritesPanel.vue                      (~120行，收藏列表)
  ├── CommentHistory.vue                      (~100行，评论历史)
  ├── QuizHistoryPanel.vue                    (~100行，答题记录)
  ├── PlantGameHistory.vue                    (~100行，植物游戏记录)
  ├── BrowseHistoryPanel.vue                  (~100行，浏览历史)
  └── ChatHistoryPanel.vue                    (~100行，AI对话历史)
```

**各组件共享的逻辑提取为 composable：**

```js
// src/composables/usePersonalCenter.js
export function usePersonalCenter() {
  const statsLoading = ref(false)
  const activeTab = ref('stats')
  const studyStats = computed(() => [...])
  // 共享分页、加载状态等
  return { statsLoading, activeTab, studyStats }
}
```

**样式处理：** 540 行 `<style scoped>` 分散到各子组件，每个组件只带自己的样式（<80行/组件）。共享样式变量/混入通过 SCSS 变量自动注入（项目已配置 Vite CSS 预处理器自动注入）。

**预期效果：** 1131 → 每文件 100-150 行，总计行数不变但可维护性大幅提升。

#### 1.2 AiChatCard.vue (916行 → 目标 <350行)

脚本逻辑 414 行，是整个聊天系统的核心。应拆分为：

```
src/components/business/display/
  ├── AiChatCard.vue                          (~200行，聊天卡片容器)
  ├── ai-chat/
  │   ├── SessionDrawer.vue                   (~120行，历史会话抽屉)
  │   ├── ChatMessageList.vue                 (~150行，消息列表+自动滚动)
  │   ├── ChatInputArea.vue                   (~100行，输入框+发送按钮)
  │   └── SessionBadge.vue                    (~50行，当前会话标签)
```

**WebSocket + 会话管理逻辑提取为 composable：**

```js
// src/composables/useAiChat.js（从 AiChatCard.vue script 中提取）
export function useAiChat() {
  // WebSocket 连接管理
  // 消息收发、流式渲染
  // 会话创建/切换/删除
  // 历史消息加载
}
```

**预期效果：** 916 → 每文件 50-200 行，逻辑可单独测试。

### 方案二：全局 CSS 拆分与去重（3158行 → 目标 <1500行）

#### 2.1 当前问题

4 个大型 CSS 文件合计 3158 行：

| 文件 | 行数 | 问题 |
|------|------|------|
| `components.css` | 951 | 组件样式散落在全局文件中 |
| `home.css` | 911 | 首页样式未按区块拆分 |
| `pages.css` | 807 | 页面样式混杂 |
| `common.css` | 489 | 通用样式应该按功能拆分 |

#### 2.2 优化策略

**短期（低风险）：CSS 文件按模块拆分**

```
src/styles/
  ├── common/
  │   ├── variables.css          (CSS 变量定义)
  │   ├── reset.css              (浏览器重置)
  │   ├── typography.css         (字体排版)
  │   └── utilities.css          (工具类)
  ├── components/
  │   ├── buttons.css
  │   ├── cards.css
  │   ├── forms.css
  │   ├── tabs.css
  │   ├── dialog.css
  │   └── skeleton.css
  └── pages/
      ├── home-hero.css
      ├── home-featured.css
      ├── knowledge.css
      └── ...
```

**中期（推荐）：样式内聚到组件**

大原则：**样式跟组件走，不跟全局文件走。**

- 将 `home.css` 中样式迁移到 `Home.vue` 及其子组件的 `<style scoped>` 中
- 将 `components.css` 中的按钮/卡片/表单样式迁移到对应的业务组件
- 只保留 `variables.css`、`reset.css`、`typography.css` 等基础全局样式在 `styles/` 目录

**长期（可选）：引入 PurgeCSS 删除未使用样式**

```bash
npm install -D @fullhuman/postcss-purgecss
```

Vite 配置示例：
```js
// vite.config.js
import purgecss from '@fullhuman/postcss-purgecss'

export default {
  css: {
    postcss: {
      plugins: [
        purgecss({
          content: ['./index.html', './src/**/*.{vue,js}'],
          safelist: [/el-/, /hljs/]  // 保留 Element Plus 和 highlight.js 类名
        })
      ]
    }
  }
}
```

### 方案三：脚本逻辑提取为 Composables

分析发现多个视图存在可以直接提取为 composable 的内联逻辑：

| 文件 | 脚本行数 | 可提取的 Composable |
|------|----------|---------------------|
| `AiChatCard.vue` | 414 | `useAiChat.js` — WebSocket+会话管理 |
| `PersonalCenter.vue` | 340 | `usePersonalCenter.js` — 用户中心数据管理 |
| `GlobalSearch.vue` | 283 | `useGlobalSearch.js` — 全局搜索逻辑 |
| `Admin.vue` | 219 | `useAdminData.js` — 已存在，需扩展 |

**Composable 提取的好处：**
- 可在多个组件间复用
- 可独立进行单元测试（纯逻辑不依赖 DOM）
- 减少 SFC 文件体积 30-50%

## 拆分模式参考

### Vue 组件拆分模板

**拆分前（巨石组件）：**
```
BigView.vue (800+ lines)
  ├── <template> — 4-5个功能区块混在一起
  ├── <script> — 所有逻辑在一个 setup() 中
  └── <style scoped> — 300+ 行样式
```

**拆分后：**
```
BigView.vue (100-150 lines, 纯编排)
  ├── <template> — 用子组件拼装页面
  ├── <script> — 只做数据获取和分发
  └── <style scoped> — <50行，仅页面级布局

big-view/
  ├── FeaturePanelA.vue     (<150行)
  ├── FeaturePanelB.vue     (<150行)
  ├── FeaturePanelC.vue     (<150行)
  └── composables/
      └── useBigView.js     (<200行，共享逻辑)
```

### Props/Events 约定

子组件通过 props 接收数据，通过 emits 回传事件：

```vue
<!-- 父组件 -->
<FeaturePanelA
  :data="featureAData"
  :loading="loading"
  @action="handleFeatureAction"
/>

<!-- 子组件 -->
<script setup>
const props = defineProps({
  data: { type: Array, required: true },
  loading: { type: Boolean, default: false }
})
const emit = defineEmits(['action'])
</script>
```
