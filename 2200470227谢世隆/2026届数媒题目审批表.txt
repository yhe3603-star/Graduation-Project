5.课题简介：
侗乡医药是广西三江侗族自治县自治区级非遗，长期依赖口传心授，缺乏标准化数字整理，非遗活态传承面临挑战。本课题从非遗保护视角出发，以数字媒体技术为支撑，设计并实现一款轻量化数字展示平台，解决 “非遗资源散落、传承渠道狭窄” 的痛点。研究内容涵盖侗乡医药非遗核心数据梳理、前后端分离架构搭建、四大核心模块（非遗知识库、传承人风采、药用资源图鉴、非遗问答）三大拓展模块（文化互动专区模块、学习资源下载模块、数据可视化模块）开发与侗族非遗元素界面融合。采用 Vue+SpringBoot+MySQL 技术栈，按软件工程流程完成全流程开发，成果可为侗乡医药非遗提供可持续的数字化传播与保护平台，为小众非遗数字化保护提供本科级解决方案，同时锻炼学生 Web 全栈开发与非遗文化可视化能力。






二、毕业设计作品的内容（具体列出要求与数据）
基础核心模块
非遗医药知识库模块：构建 “疗法分类 + 病症分类 + 药材分类” 三级检索体系，涵盖 12 项经典药方、6 类核心疗法的详细内容，详情页需整合文字解析、步骤示意图、操作视频及相关药材关联信息，同时设置知识更新日志，保障信息时效性与准确性。
传承人风采模块：按 “国家级 / 自治区级 / 县级” 传承级别分级展示 8 名传承人信息，详情页需集成从业履历、技艺特色、代表案例及访谈视频，预留传承人动态更新入口，便于后续内容迭代。
药用资源图鉴模块：搭建 “草本 / 木本”“外用 / 内服” 双维度筛选体系，为 15 种常用药用植物配备高清实拍图、学名、侗语名、生长环境及药用故事，实现图文联动浏览，直观呈现侗族药用植物资源。
非遗医药问题解答模块：按 “知识理解 / 使用咨询 / 文化背景” 分类整理问答内容，开发关键词模糊搜索功能，增设用户问题反馈表单，及时响应用户疑问并为内容优化提供依据。
拓展模块
文化互动专区模块：设置侗乡非遗医药知识趣味答题、药用植物识别小游戏等轻度互动功能，答题题库需包含至少 50 道基础题与 20 道进阶题；同时搭建用户评论区，支持用户对知识内容、传承人故事进行留言交流，增强平台互动性与用户粘性。
学习资源下载模块：整合侗乡非遗医药相关的学术论文摘要、科普手册、药方图解等学习素材，提供 PDF 格式下载服务，且需按 “入门 / 进阶 / 专业” 分类，满足学生群体及医药爱好者的学习拓展需求。
数据可视化模块：通过图表直观呈现药用植物地域分布、药方使用频次等数据，支持数据筛选与查看详情，帮助用户从宏观层面理解侗乡医药的文化与资源体系。

命题内容和要求：
选题背景
侗乡医药作为三江侗族自治县自治区级非物质文化遗产，承载着侗族独特的诊疗理念、药用资源与炮制技艺，现有 10 名各级传承人，其鼻炎疗法、药浴疗法等传统智慧兼具文化价值与实用价值。从非遗保护视角来看，当前侗乡医药面临传承危机：知识体系依赖口传心授，缺乏标准化数字化归档；传播渠道狭窄，公众认知度低，年轻群体接触途径匮乏，非遗活态传承受阻。结合数字媒体技术专业实践导向与非遗数字化保护需求，本选题聚焦 “非遗活态传承” 核心目标，设计一款非遗视角下的侗乡医药数字展示平台，通过标准化数据整理、沉浸式展示与便捷传播，既契合本科毕业设计能力范畴，又为小众非遗提供可复制的数字化保护方案，助力侗乡医药这一非遗资源走进公众视野，具有重要的实践意义与文化价值。

任务要求
需求分析：需求分析：遵循软件工程规范，结合非遗保护相关标准，调研普通公众、非遗研究者、医药文化爱好者等核心用户需求；系统梳理侗乡医药核心资源，涵盖 10 项以上经典药方、5 类核心疗法（含鼻炎疗法、药浴疗法等）、传承人详细简介、药用植物图鉴等基础信息，编制逻辑清晰、可落地的需求规格说明书。
系统设计：（1）架构设计：采用前后端分离架构，简化层级设计逻辑，降低开发复杂度，保障系统稳定运行与后续维护便捷性；（2）功能模块设计：聚焦 4 大核心模块与3大拓展模块，各模块下设细分功能点，确保覆盖用户核心需求且无功能冗余：① 非遗医药知识库模块：核心功能包括 “分类检索”、“知识详情”、“收藏与分享”、 “非遗知识更新日志” 子功能，记录新增 / 修订的医药知识条目，提升内容可信度。② 传承人风采模块：包含 “传承人列表”、“个人详情页”、“传承人动态”。③ 药用资源图鉴模块：实现 “图文联动浏览”、“分类筛选”、“药材故事”，提升内容趣味性与文化性。④ 非遗医药问题解答模块：按 “知识理解类”、“使用咨询类”、“文化背景类”3 类分类展示问题；支持 “关键词搜索” 功能，用户输入问题关键词可快速匹配答案；设 “问题反馈” 入口，允许用户提交未覆盖的疑问，为后续功能优化预留空间。⑤ 文化互动专区模块（拓展）：核心功能包括 “趣味答题”、“植物识别小游戏”、“用户评论交流” 子功能，通过轻量化互动形式增强用户参与感，提升平台粘性与文化传播效果。⑥ 学习资源下载模块（拓展）：包含 “资源分类展示”、“在线预览”、“下载与收藏” 子功能，按层级整合学习素材，满足不同用户群体的学习拓展需求，助力知识深度传播。⑦ 数据可视化模块（拓展）：实现 “地域分布可视化”、“使用频次统计” 子功能，通过直观图表呈现核心数据，帮助用户从宏观层面理解侗乡医药的文化与资源体系。（3）界面设计：有机融入侗族传统元素与医药文化意象，如以侗锦经典的 “万字纹”“回纹” 作为页面边框装饰，以药用植物（如侗药常用的艾草、金银花）简约线条作为图标设计灵感；设计 5 个核心页面，各页面遵循 “简洁直观、层次分明” 原则：（4）数据库设计：基于 MySQL 设计精简高效的数据表（含医药知识表、传承人表、药用植物表、分类表），字段设计聚焦核心信息，确保数据存储规范、查询高效。

系统设计：（1）架构设计：采用前后端分离架构，简化层级设计逻辑，降低开发复杂度，保障系统稳定运行与后续维护便捷性；（2）功能模块设计：聚焦 4 大核心模块与3大拓展模块，各模块下设细分功能点，确保覆盖用户核心需求且无功能冗余：① 非遗医药知识库模块：核心功能包括 “分类检索”、“知识详情”、“收藏与分享”、 “非遗知识更新日志” 子功能，记录新增 / 修订的医药知识条目，提升内容可信度。② 传承人风采模块：包含 “传承人列表”、“个人详情页”、“传承人动态”。③ 药用资源图鉴模块：实现 “图文联动浏览”、“分类筛选”、“药材故事”，提升内容趣味性与文化性。④ 非遗医药问题解答模块：按 “知识理解类”、“使用咨询类”、“文化背景类”3 类分类展示问题；支持 “关键词搜索” 功能，用户输入问题关键词可快速匹配答案；设 “问题反馈” 入口，允许用户提交未覆盖的疑问，为后续功能优化预留空间。⑤ 文化互动专区模块（拓展）：核心功能包括 “趣味答题”、“植物识别小游戏”、“用户评论交流” 子功能，通过轻量化互动形式增强用户参与感，提升平台粘性与文化传播效果。⑥ 学习资源下载模块（拓展）：包含 “资源分类展示”、“在线预览”、“下载与收藏” 子功能，按层级整合学习素材，满足不同用户群体的学习拓展需求，助力知识深度传播。⑦ 数据可视化模块（拓展）：实现 “地域分布可视化”、“使用频次统计” 子功能，通过直观图表呈现核心数据，帮助用户从宏观层面理解侗乡医药的文化与资源体系。（3）界面设计：有机融入侗族传统元素与医药文化意象，如以侗锦经典的 “万字纹”“回纹” 作为页面边框装饰，以药用植物（如侗药常用的艾草、金银花）简约线条作为图标设计灵感；设计 5 个核心页面，各页面遵循 “简洁直观、层次分明” 原则：（4）数据库设计：基于 MySQL 设计精简高效的数据表（含医药知识表、传承人表、药用植物表、分类表），字段设计聚焦核心信息，确保数据存储规范、查询高效。
系统开发：（1）前端开发：运用 HTML、CSS、JavaScript 语言及 Vue 框架，实现页面静态展示与基础交互功能，包括分类筛选、页面平滑跳转、图文 / 短视频正常播放等；（2）后端开发：采用 Java 语言与 Spring Boot 框架，完成基础接口开发，实现数据查询、列表展示、详情返回等核心功能；（3）数据库实现：完成数据表创建与优化，精准录入梳理后的侗乡医药核心数据，实现数据的基础增删改查操作；（4）功能集成：完成前后端高效对接，确保页面数据加载正常、各功能模块稳定运行，解决主流浏览器兼容等基础问题。
系统测试与优化：制定标准化测试方案，开展全面功能测试（确保所有页面可正常访问、数据无偏差）与易用性测试，针对性优化页面加载速度与操作流畅度，保障系统满足基础使用需求。
5. 文档撰写：严格按照学校要求完成开题报告、英文翻译材料、毕业设计论文的撰写工作，论文需详细阐述系统设计与开发全流程，附核心页面截图、关键代码说明及完整测试报告。

系统设计：（1）架构设计：采用前后端分离架构，简化层级设计逻辑，降低开发复杂度，保障系统稳定运行与后续维护便捷性；（2）功能模块设计：聚焦 4 大核心模块与3大拓展模块，各模块下设细分功能点，确保覆盖用户核心需求且无功能冗余：① 非遗医药知识库模块：核心功能包括 “分类检索”、“知识详情”、“收藏与分享”、 “非遗知识更新日志” 子功能，记录新增 / 修订的医药知识条目，提升内容可信度。② 传承人风采模块：包含 “传承人列表”、“个人详情页”、“传承人动态”。③ 药用资源图鉴模块：实现 “图文联动浏览”、“分类筛选”、“药材故事”，提升内容趣味性与文化性。④ 非遗医药问题解答模块：按 “知识理解类”、“使用咨询类”、“文化背景类”3 类分类展示问题；支持 “关键词搜索” 功能，用户输入问题关键词可快速匹配答案；设 “问题反馈” 入口，允许用户提交未覆盖的疑问，为后续功能优化预留空间。⑤ 文化互动专区模块（拓展）：核心功能包括 “趣味答题”、“植物识别小游戏”、“用户评论交流” 子功能，通过轻量化互动形式增强用户参与感，提升平台粘性与文化传播效果。⑥ 学习资源下载模块（拓展）：包含 “资源分类展示”、“在线预览”、“下载与收藏” 子功能，按层级整合学习素材，满足不同用户群体的学习拓展需求，助力知识深度传播。⑦ 数据可视化模块（拓展）：实现 “地域分布可视化”、“使用频次统计” 子功能，通过直观图表呈现核心数据，帮助用户从宏观层面理解侗乡医药的文化与资源体系。（3）界面设计：有机融入侗族传统元素与医药文化意象，如以侗锦经典的 “万字纹”“回纹” 作为页面边框装饰，以药用植物（如侗药常用的艾草、金银花）简约线条作为图标设计灵感；设计 5 个核心页面，各页面遵循 “简洁直观、层次分明” 原则：（4）数据库设计：基于 MySQL 设计精简高效的数据表（含医药知识表、传承人表、药用植物表、分类表），字段设计聚焦核心信息，确保数据存储规范、查询高效。
系统开发：（1）前端开发：运用 HTML、CSS、JavaScript 语言及 Vue 框架，实现页面静态展示与基础交互功能，包括分类筛选、页面平滑跳转、图文 / 短视频正常播放等；（2）后端开发：采用 Java 语言与 Spring Boot 框架，完成基础接口开发，实现数据查询、列表展示、详情返回等核心功能；（3）数据库实现：完成数据表创建与优化，精准录入梳理后的侗乡医药核心数据，实现数据的基础增删改查操作；（4）功能集成：完成前后端高效对接，确保页面数据加载正常、各功能模块稳定运行，解决主流浏览器兼容等基础问题。
系统测试与优化：制定标准化测试方案，开展全面功能测试（确保所有页面可正常访问、数据无偏差）与易用性测试，针对性优化页面加载速度与操作流畅度，保障系统满足基础使用需求。
5. 文档撰写：严格按照学校要求完成开题报告、英文翻译材料、毕业设计论文的撰写工作，论文需详细阐述系统设计与开发全流程，附核心页面截图、关键代码说明及完整测试报告。

1．本课题的研究内容、重点及难点
研究内容
本课题以三江侗族自治县自治区级非物质文化遗产 —— 侗乡医药为核心，从非遗活态传承视角出发，紧扣数字媒体技术专业实践能力范畴，开发一款集非遗展示、知识普及、文化传播于一体的数字展示平台，为侗乡医药非遗搭建标准化、可持续的线上传播与保护载体，具体研究内容分为 5 个核心维度：
前期调研与数据梳理
文献与官方资料检索：通过查阅地方文献，以及三江县文旅局官网 “非遗保护” 专栏的公开资料，系统收集侗乡医药的经典药方、核心疗法、传承人信息、药用植物图鉴等基础资源；同时检索非遗数字化保护、轻量化网站开发相关的学术文献，为课题提供理论与技术支撑。民间资料采集：借助三江县文化馆非遗展厅线上 VR 展、侗族医药传承人工坊公开访谈视频等渠道，补充传承人临床实践案例、药用植物采集习俗故事等文化性内容，丰富网站的内容层次。用户需求挖掘：设计面向普通公众、医药文化爱好者、在校学生等核心用户的调查问卷，结合对 “中国非物质文化遗产网”“贵州苗医药数字化平台” 等同类型竞品网站的分析，明确不同用户群体的知识获取偏好与功能需求，为后续开发奠定数据与需求基础。
需求分析与方案制定
严格遵循软件工程规范，划分用户需求、功能需求、非功能需求三类核心需求，结合非遗保护标准，明确平台 “非遗传承优先、文化展示为主、实用功能为辅” 的设计定位，编制需求规格说明书。
系统设计
架构设计采用前后端分离架构，前端基于 Vue CLI 构建框架，后端依托 Spring Boot 搭建，通过 RESTful API 规范设计接口，简化架构复杂度。功能模块设计包含基础核心模块与拓展模块，界面设计融入侗族非遗元素与医药意象，数据库基于 MySQL 8.0 设计核心数据表，绘制 ER 图明确数据关联。
功能模块设计：
基础核心模块：
非遗医药知识库模块：检索功能：支持 “疗法分类（如鼻炎疗法、药浴疗法等 6 类）+ 病症分类（如呼吸道疾病、跌打损伤等）+ 药材分类（如草本、木本等）” 三级联动检索，同时提供关键词精准搜索与模糊匹配功能，搜索结果支持按 “热度 / 更新时间” 排序。内容展示：涵盖 12 项经典药方（如百草汤配方、鼻炎外用方等）、6 类核心疗法的完整内容，详情页包含 4 个子板块 —— 文字解析（含配方组成、适用病症、使用禁忌）、步骤示意图（关键操作节点可视化）、操作视频（3-8 分钟精简版，支持倍速播放、暂停截图）、相关药材关联（直接跳转至药用资源图鉴对应条目）。辅助功能：设置知识更新日志，记录每条内容的首次上线时间、修订版本、修订原因；支持 “收藏 / 取消收藏” 功能，用户可在个人中心查看收藏列表；提供 “纠错反馈” 按钮，用户可提交内容误差建议。
传承人风采模块：分级展示：按 “国家级 / 自治区级 / 县级” 分类陈列 8 名传承人卡片，卡片显示姓名、传承级别、核心技艺、从业年限等关键信息，支持按 “传承级别 / 从业年限” 筛选排序。详情页内容：集成 5 大核心信息 —— 从业履历（含入门时间、师从关系、重要从业节点）、技艺特色（核心擅长领域、独特诊疗手法描述）、代表案例（2-3 个典型临床案例，含病情概述、诊疗过程、效果反馈）、访谈视频（5-10 分钟精简版，支持全屏播放、音量调节）、荣誉资质（获奖证书、官方认定文件等图片展示）。动态更新：预留传承人动态专栏入口，可后续补充传承人最新活动、技艺传承成果、新增案例等内容，支持管理员后台快速上传更新。
药用资源图鉴模块：筛选功能：搭建 “植物类型（草本 / 木本）”“使用方式（外用 / 内服）” 双维度筛选体系，支持单一维度筛选或组合筛选，筛选结果实时加载。图鉴内容：为 15 种常用药用植物提供标准化展示 —— 高清实拍图（含整体植株、药用部位特写图各 1-2 张）、基础信息（学名、侗语名、科属、药用部位）、生长环境（气候要求、地理分布区域）、药用价值（核心功效、配伍应用场景）、药用故事（民间采集习俗、传承中的趣味典故）。浏览体验：实现图文联动，点击图片可放大查看细节，支持左右滑动切换图鉴条目；提供 “相似药材推荐” 功能，基于药用功效或植物特征关联展示相关条目。
非遗医药问题解答模块：分类展示：按 “知识理解（如药方原理、疗法适用范围等）”“使用咨询（如药材获取、操作注意事项等）”“文化背景（如医药起源、传承历史等）” 三大类整理问答，每类包含 15-20 条核心问题，按用户关注度排序。搜索功能：支持关键词模糊搜索，输入关键词后实时匹配相关问题，显示匹配度排序结果；提供热门搜索关键词推荐，展示用户高频查询内容。反馈互动：增设用户问题反馈表单，表单包含 “问题分类、问题描述、联系方式（可选）” 字段，用户提交后后台可接收并处理；设置 “问题已解答” 公示区，定期更新新增问题及官方回复。
拓展模块：
文化互动专区模块：趣味答题功能：搭建答题系统，包含 50 道基础题（侧重医药基础知识、文化常识）、20 道进阶题（侧重深度知识、案例分析）；支持随机组卷（每次 10 题，基础题 7 道 + 进阶题 3 道）、指定分类答题；答题结束后即时显示分数、错题解析及正确答案，支持查看答题记录。药用植物识别小游戏：提供 15 种药用植物的高清局部图或特征图，用户需从 4 个选项中选择正确名称，游戏设置 3 个难度等级（初级 3 种植物、中级 5 种植物、高级 7 种植物）；完成游戏可获得 “识别小能手” 等电子勋章，支持分享至社交平台。用户评论交流：在知识库详情页、传承人风采页、图鉴页均设置评论区，用户登录后可发布留言，可对他人评论进行点赞、回复；评论区设置管理员审核机制，过滤违规内容，同时支持用户举报不良评论。
学习资源下载模块：资源分类：按 “入门 / 进阶 / 专业” 三级分类整理学习素材，总量不少于 30 份。资源形式：所有素材均提供 PDF 格式下载，文件大小控制在 10MB 以内，确保下载速度；部分重要素材支持在线预览（前 3 页），预览后可选择下载。辅助功能：提供资源搜索功能，支持按资源名称、关键词检索；显示资源下载量、收藏量，为用户推荐热门资源；设置 “资源更新提醒”，用户可订阅分类资源，新增资源时收到站内通知。
数据可视化模块：药用植物地域分布：以广西三江侗族自治县地图为底图，用热力图或标记点展示 15 种药用植物的主要分布区域；点击标记点可查看该区域分布的植物列表及每种植物的生长环境详情。药方使用频次统计：以柱状图展示 12 项经典药方的历史使用频次、地域使用热度；支持按 “年度 / 季度” 筛选数据，可切换图表类型（柱状图 / 折线图），点击图表条目可跳转至对应药方详情页。
数据更新：支持管理员后台上传最新数据，图表实时同步更新；提供数据下载功能，支持导出 Excel 格式统计数据，满足学生、研究人员的分析需求。
界面设计：将侗族文化元素与医药意象深度融合，主色调选取侗锦靛蓝色与药用植物草绿色，以侗锦 “万字纹”“回纹” 装饰页面边框，以艾草、金银花等药用植物线条设计功能图标，完成首页、知识库列表页等 5 个核心页面的响应式原型设计，兼顾美观性与操作便捷性。
数据库设计：基于 MySQL 8.0 设计轻量化数据库，构建医药知识表、传承人表、药用植物表等 6 张核心数据表，绘制 ER 图明确数据关联，为高频查询字段建立索引，保障数据存储规范与查询高效。
④　系统开发与调试
前端开发：运用 HTML5 语义化标签搭建页面结构，通过 CSS3 Flex/Grid 布局实现响应式适配，结合 JavaScript 与 Vue 框架开发交互逻辑；集成 Video.js 插件实现短视频播放，通过 localStorage 实现收藏功能，开发社交平台分享接口，完成多终端页面调试。后端开发：基于 Java 与 Spring Boot 框架开发核心接口，实现数据查询、列表分页、详情返回、用户行为记录等功能；为接口添加参数校验与异常处理，确保返回标准化 JSON 格式数据，完成接口联调保障数据传输精准。数据库部署：编写 SQL 脚本创建数据表，录入前期梳理的 12 项药方、6 类疗法、8 名传承人等核心数据，测试数据增删改查功能，确保数据一致性与准确性。前后端对接与调试：完成前后端功能集成，测试模块联动效果；通过 Autoprefixer 处理 CSS 兼容、Babel 转译 JS 代码，解决主流浏览器兼容问题；开展初步功能测试，修复接口调用、数据展示等基础 bug。
⑤　测试优化与文档撰写
多维度测试：制定覆盖全功能点的测试用例，开展功能测试验证功能完整性与数据准确性、性能测试模拟并发访问优化加载速度、易用性测试邀请目标用户试用并收集优化建议。迭代优化：针对测试中发现的关键词搜索精准度不足、移动端图片加载卡顿等问题，进行代码修订与功能优化，确保系统满足预设指标。配套文档撰写：严格按照学校要求，完成开题报告、不少于 2 万字符的英文翻译材料、1.5 万汉字以上的毕业设计论文撰写，论文需详细阐述开发全流程并附核心页面截图、关键代码及测试报告。
研究重点
侗乡非遗医药数据的权威性与完整性以三江非遗官方发布的资料为核心依据，对收集的药方、疗法、传承人信息等数据进行多渠道交叉核验，确保数据来源正规、内容准确，避免出现信息偏差，为网站内容的可信度提供保障。
非遗视角下功能模块的实用性与轻量化平衡基于前期用户调研结果，在 4 大核心模块与3大拓展模块设计中精准匹配用户需求，优先保留知识查询、收藏分享、分类筛选等高频功能，剔除数据统计分析、用户等级管理等非必要复杂功能；同时严格把控开发难度，确保技术选型与功能设计契合本科阶段数字媒体技术专业的知识与能力范畴，实现 “实用功能全覆盖、开发难度可控” 的平衡。
前后端数据交互的稳定性重点保障前端交互操作与后端接口响应的精准对接，在接口开发阶段添加完善的异常处理机制，避免因网络波动、参数错误导致的数据传输失败；通过多次接口联调与压力测试，优化数据请求逻辑，确保知识检索、详情加载、收藏反馈等核心功能无延迟、无错误，提升用户使用体验。
研究难点
小众非遗数据的获取与核验侗乡医药作为区域性小众非遗，部分核心药方、独特疗法仅留存于民间传承人的口述经验或未公开的地方文献中，公开渠道可获取的资料有限，存在数据收集不全面的问题；同时，民间资料缺乏统一标准，不同来源的信息可能存在冲突，需通过走访传承人、咨询当地非遗保护部门、多文献交叉比对等方式核验数据真实性与准确性，数据收集与核验的难度较高。
轻量化架构下的功能完整性为契合本科毕业设计的开发能力，课题需采用简化的轻量化技术架构，这对功能设计的精准性提出了更高要求。需在剔除复杂架构与冗余功能的前提下，确保知识查询、分类检索、收藏分享、关键词搜索等核心功能完整实现，既要避免因架构简化导致功能缺失，又要防止为保障功能完整而增加开发复杂度，需精准把控技术与功能的平衡，兼顾开发效率与用户体验。

3. 实施方案、进度实施计划及预期提交的毕业设计资料
实施方案
需求分析阶段（第1到第2周）
需求细化：结合前期调研，明确三类核心需求：
用户需求：普通公众（知识查询、文化浏览）、医药爱好者（深度药方解析、传承人信息）、学生（学习素材下载、知识拓展）；
功能需求：确定医药知识库、传承人风采、药用资源图鉴、常见问题解答 4 大模块的具体功能点，形成功能需求清单；
非功能需求：明确页面加载速度≤3 秒、兼容 Chrome/Firefox/Edge/Safari 主流浏览器、移动端适配分辨率≥720×1280、数据查询响应时间≤0.5 秒等指标。
文档输出：编制开题报告与任务书，包含需求概述、用户画像、功能清单、非功能指标、数据规范等内容，经指导教师审核后定稿。
系统设计阶段（第3到第5周）
架构设计：采用前后端分离架构，前端基于 Vue CLI 搭建项目框架，通过 Vue Router 实现路由管理、Axios 实现数据请求；后端基于 Spring Boot 框架构建，通过 MyBatis-Plus 实现数据库交互，采用 RESTful API 规范设计接口，简化中间件依赖，降低架构复杂度。
功能模块设计：对 4 大核心模块进行精细化设计，绘制模块功能流程图：
医药知识库：设计 “疗法分类 + 病症分类 + 药材分类” 三级检索体系，知识详情页包含 “文字解析 + 步骤示意图 + 操作视频 + 相关药材关联”，同时设 “知识更新日志” 记录数据修订情况；
传承人风采：按 “国家级 / 自治区级 / 县级” 分级展示传承人列表，个人详情页整合 “从业履历 + 技艺特色 + 代表案例 + 访谈视频”，并预留 “传承人动态” 更新入口；
药用资源图鉴：构建 “草本 / 木本”“外用 / 内服” 双维度筛选体系，每个药材条目包含 “高清实拍图 + 名称 + 生长环境 + 药用故事”，实现图文联动浏览；
常见问题解答：按 “知识理解 / 使用咨询 / 文化背景” 分类整理问答，开发关键词模糊搜索功能，增设用户问题反馈表单，为后续内容迭代预留空间。
界面设计：
风格定位：以 “侗族文化 + 医药清新” 为核心风格，主色调选取侗锦中的靛蓝色（#1A5276）与药用植物的草绿色（#28B463）；
元素融合：将侗锦 “万字纹”“回纹” 作为页面边框装饰，以艾草、金银花等药用植物线条设计功能图标，首页 Banner 融入侗族鼓楼与药田的插画；
页面布局：完成 5 个核心页面的原型设计（首页、知识库列表页、知识详情页、传承人页、药材图鉴页），确保响应式适配，移动端优化触控按钮尺寸（≥44×44px）。
数据库设计：基于 MySQL 8.0 设计核心数据表，绘制 ER 图，确保数据关联性与查询效率：如医药知识表；传承人表；药用植物表；常见问题表；分类表；用户行为表。
系统开发阶段（第5到第11周）
前端开发：运用 HTML5 语义化标签搭建页面结构，CSS3 Flex/Grid 布局实现响应式适配，JavaScript 结合 Vue 框架开发交互逻辑；
集成 Video.js 插件实现视频播放，通过 localStorage 实现用户收藏功能，开发分享接口对接主流社交平台；
完成页面调试，确保各终端界面布局一致、交互流畅。
后端开发：基于 Java 与 Spring Boot框架开发接口，实现数据查询、列表分页、详情返回、用户行为记录等核心功能；
接口添加参数校验与异常处理，返回标准化 JSON 格式数据；
完成接口联调，确保前端请求可精准获取后端数据。
数据库开发：编写 SQL 脚本创建数据表，录入前期梳理的侗乡医药核心数据（药方、疗法、传承人、药用植物、问答）；
为高频查询字段（如 knowledge_id、name、keywords）建立索引，优化查询效率；
测试数据增删改查功能，确保数据一致性。
功能集成：完成前后端对接，测试模块联动功能（如知识库药材链接跳转至图鉴页）；进行初步功能测试，修复接口调用、数据展示等基础 bug。
测试优化阶段（第12到13周）
测试方案制定：设计覆盖所有功能点的测试用例，包含正向用例（如 “筛选鼻炎疗法返回对应知识”）与异常用例（如 “搜索无结果时提示‘暂无相关内容’”）。
多维度测试：
功能测试：通过测试验证所有功能点的完整性与准确性，确保无功能缺失、数据无错误；
性能测试：使用 JMeter 模拟 100 人并发访问，测试页面加载速度与接口响应时间，通过压缩图片、合并静态资源优化性能；
易用性测试：邀请 8 名目标用户试用，通过用户反馈优化操作流程（如简化筛选步骤、调整按钮位置）。
迭代优化：针对测试中发现的问题进行代码修订，确保系统满足预设指标。
文档撰写阶段（第14到15周）
论文撰写：按学校规范完成 1.5 万汉字以上的毕业论文与论文英文翻译，涵盖 “课题简介及选题背景、设计调研、设计定位、设计过程、设计成果、设计总结、设计展望”7大章节，附核心页面截图、关键代码、测试报告；
资料整理：汇总所有设计图纸、源代码、测试文档，形成完整的毕业设计资料包。
进度实施计划
第1-2周：开题启动与基础资料分析。资料收集与调研，精读文献，确定技术路线。撰写并提交任务书、开题报告及外文翻译材料。
第3-4周；原型与风格定稿。确定网站整体风格、功能模块，并完成高保真UI/UX设计原型图。参加开题答辩，根据意见修改报告。
第5-6周：前端架构与核心交互。搭建Vue3前端项目框架，完成非遗医药知识库、传承人风采等静态页面及基础交互。
第7-8周：后端与数据库搭建。搭建SpringBoot3后端项目，设计数据库结构。完成后端基础架构与非遗医药知识库、传承人等核心数据表的创建。
第9-11周：核心功能实现。实现网站前后端数据交互接口、保障前后端数据一致准确。继续完善4大核心模块与3大拓展模块。
